## Track TikTok Videos Programmatically

This guide shows you how to track TikTok video performance using the ContentStats.io API. Get hourly snapshots of views, likes, comments, shares, and saves with built-in bot detection.

<Note>
  **No TikTok API approval required.** Start tracking in minutes with just an API key.
</Note>

## Prerequisites

Before you start, you'll need:

- ContentStats.io account ([sign up free](https://contentstats.io/auth/signup))
- API key from your [dashboard](https://contentstats.io/dashboard/api-keys)
- At least $0.015 in your balance (you get $5 free)
- URL of a **public** TikTok video

## Step 1: Get Your API Key

<Steps>
  <Step title="Access Dashboard">
    Log in to [contentstats.io](https://contentstats.io/dashboard)
  </Step>
  
  <Step title="Navigate to API Keys">

  </Step>

</Warning>

## Step 2: Start Tracking a TikTok Video

Send a POST request to `/api/v1/videos/track`:

<CodeGroup>
  ```bash cURL
  curl -X POST https://contentstats.io/api/v1/videos/track \
    -H "Content-Type: application/json" \
    -H "X-API-Key: cs_live_YOUR_KEY" \
    -d '{
      "video_link": "https://www.tiktok.com/@charlidamelio/video/7234567890",
      "duration_days": 7
    }'
  ```

  ```javascript Node.js
  const response = await fetch('https://contentstats.io/api/v1/videos/track', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': process.env.CONTENTSTATS_API_KEY
    },
    body: JSON.stringify({
      video_link: 'https://www.tiktok.com/@charlidamelio/video/7234567890',
      duration_days: 7
    })
  });

  const data = await response.json();
  console.log('Tracking started:', data.id);
  ```

  ```python Python
  import requests
  import os

  response = requests.post(
      'https://contentstats.io/api/v1/videos/track',
      headers={
          'Content-Type': 'application/json',
          'X-API-Key': os.environ['CONTENTSTATS_API_KEY']
      },
      json={
          'video_link': 'https://www.tiktok.com/@charlidamelio/video/7234567890',
          'duration_days': 7
      }
  )

  data = response.json()
  print(f"Tracking started: {data['id']}")
  ```

  ```php PHP
  <?php
  $curl = curl_init();

  curl_setopt_array($curl, [
      CURLOPT_URL => 'https://contentstats.io/api/v1/videos/track',
      CURLOPT_RETURNTRANSFER => true,
      CURLOPT_POST => true,
      CURLOPT_HTTPHEADER => [
          'Content-Type: application/json',
          'X-API-Key: ' . getenv('CONTENTSTATS_API_KEY')
      ],
      CURLOPT_POSTFIELDS => json_encode([
          'video_link' => 'https://www.tiktok.com/@charlidamelio/video/7234567890',
          'duration_days' => 7
      ])
  ]);

  $response = curl_exec($curl);
  $data = json_decode($response, true);
  echo "Tracking started: " . $data['id'];
  ?>
  ```
</CodeGroup>

## Understanding Bot Analysis

Each video gets AI-powered bot detection:

| Score | Meaning | Action |
|-------|---------|--------|
| **0.0 - 0.3** | Authentic | ‚úÖ Engagement looks real |
| **0.3 - 0.5** | Suspicious | ‚ö†Ô∏è Some red flags detected |
| **0.5 - 0.7** | Likely Botted | üö® High probability of fake engagement |
| **0.7 - 1.0** | Definitely Botted | ‚ùå Clear bot activity |

<Accordion title="What each score measures">
  - **Engagement Velocity**: Speed of likes/comments (sudden spikes = bots)
  - **Comment Quality**: Generic/spammy comments indicate bots
  - **Timing Patterns**: 24/7 engagement without dips = suspicious
  - **Ratio Analysis**: Unusual views:likes:comments ratios
  - **Channel Context**: Account age, follower count, history
</Accordion>

## Step 4: List All Tracked TikTok Videos

Get all your tracked videos:

```bash
curl "https://contentstats.io/api/v1/videos?platform=tiktok&status=monitoring" \
  -H "X-API-Key: cs_live_YOUR_KEY"
```

### Query Parameters

| Parameter | Values | Description |
|-----------|--------|-------------|
| `platform` | `tiktok`, `youtube`, `instagram`, `twitter` | Filter by platform |
| `status` | `monitoring`, `completed`, `paused`, `error` | Filter by status |
| `limit` | 1-100 | Number of results (default: 50) |

## Step 5: Stop Tracking Early (Optional)

To stop tracking before the duration ends:

```bash
curl -X POST https://contentstats.io/api/v1/videos/cm5abc123def/stop \
  -H "X-API-Key: cs_live_YOUR_KEY"
```

<Info>
  You'll only pay for snapshots already collected. Future snapshots won't be charged.
</Info>

## Advanced: Batch Tracking

Track multiple TikTok videos at once:

```javascript
const videos = [
  'https://www.tiktok.com/@user1/video/123',
  'https://www.tiktok.com/@user2/video/456',
  'https://www.tiktok.com/@user3/video/789'
];

const results = await Promise.all(
  videos.map(video_link =>
    fetch('https://contentstats.io/api/v1/videos/track', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-API-Key': process.env.CONTENTSTATS_API_KEY
      },
      body: JSON.stringify({ video_link, duration_days: 7 })
    }).then(r => r.json())
  )
);

console.log(`Started tracking ${results.length} videos`);
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: Video not found">
    **Cause**: Invalid URL or video is private/deleted
    
    **Fix**: Verify the URL is public and accessible without login
  </Accordion>
  
  <Accordion title="Error: Insufficient balance">
    **Cause**: Not enough credit for estimated cost
    
    **Fix**: [Add credits](https://contentstats.io/dashboard/billing) to your account
  </Accordion>
  
  <Accordion title="No snapshots appearing">
    **Cause**: First snapshot takes up to 1 hour
    
    **Fix**: Wait 60 minutes and check again
  </Accordion>
  
  <Accordion title="Duplicate tracking">
    **Cause**: Same video tracked multiple times
    
    **Fix**: List videos first to check if already tracking:
    ```bash
    curl "https://contentstats.io/api/v1/videos?platform=tiktok" \
      -H "X-API-Key: YOUR_KEY"
    ```
  </Accordion>
</AccordionGroup>
